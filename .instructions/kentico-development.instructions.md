## Content Type Models

### Generated Models Pattern

All content type models are **auto-generated** from Kentico content types. They follow this pattern:

#### Web Page Content Type
```csharp
//--------------------------------------------------------------------------------------------------
// <auto-generated>
//     This code was generated by code generator tool.
// </auto-generated>
//--------------------------------------------------------------------------------------------------

using System;
using System.Collections.Generic;
using CMS.ContentEngine;
using CMS.Websites;

namespace DancingGoat.Models
{
    /// <summary>
    /// Represents a page of type <see cref="ArticlePage"/>.
    /// </summary>
    [RegisterContentTypeMapping(CONTENT_TYPE_NAME)]
    public partial class ArticlePage : IWebPageFieldsSource, ISEOFields
    {
        /// <summary>
        /// Code name of the content type.
        /// </summary>
        public const string CONTENT_TYPE_NAME = "DancingGoat.ArticlePage";

        /// <summary>
        /// Represents system properties for a web page item.
        /// </summary>
        [SystemField]
        public WebPageFields SystemFields { get; set; }

        /// <summary>
        /// ArticleTitle.
        /// </summary>
        public string ArticleTitle { get; set; }

        /// <summary>
        /// ArticlePagePublishDate.
        /// </summary>
        public DateTime ArticlePagePublishDate { get; set; }

        /// <summary>
        /// ArticlePageTeaser.
        /// </summary>
        public IEnumerable<Image> ArticlePageTeaser { get; set; }

        // Additional properties...
    }
}
```

#### Reusable Content Type
```csharp
[RegisterContentTypeMapping(CONTENT_TYPE_NAME)]
public partial class Image : IContentItemFieldsSource
{
    public const string CONTENT_TYPE_NAME = "DancingGoat.Image";

    [SystemField]
    public ContentItemFields SystemFields { get; set; }

    public ContentItemAsset ImageFile { get; set; }
    public string ImageShortDescription { get; set; }
}
```

### Key Points:
- Models are **partial classes** - extend them in separate files for custom logic
- Use `IWebPageFieldsSource` for web pages
- Use `IContentItemFieldsSource` for reusable content
- Always include `[RegisterContentTypeMapping(CONTENT_TYPE_NAME)]` attribute
- System properties accessed via `SystemFields` property
- Asset fields use `ContentItemAsset` type

## Widget Development

### Widget File Structure

Each widget consists of 3-4 files in a dedicated folder:

```
/Components/Widgets/CardWidget/
  CardWidgetProperties.cs      - Widget properties class
  CardWidgetViewComponent.cs   - Widget logic (optional)
  CardWidgetViewModel.cs       - View model for the widget
  _CardWidget.cshtml           - Widget view
```

### Widget Properties

Properties define the configurable fields in the widget configuration dialog.

```csharp
using Kentico.PageBuilder.Web.Mvc;
using Kentico.Xperience.Admin.Base.FormAnnotations;
using CMS.ContentEngine;

namespace DancingGoat.Widgets
{
    /// <summary>
    /// Card widget properties.
    /// </summary>
    public class CardWidgetProperties : IWidgetProperties
    {
        /// <summary>
        /// Image to be displayed.
        /// </summary>
        [ContentItemSelectorComponent(
            Models.Image.CONTENT_TYPE_NAME, 
            Label = "{$dancinggoat.cardwidget.image.label$}", 
            Order = 1)]
        public IEnumerable<ContentItemReference> Image { get; set; } = new List<ContentItemReference>();

        /// <summary>
        /// Text to be displayed.
        /// </summary>
        public string Text { get; set; }
    }
}
```

### Common Property Attributes:
- `[ContentItemSelectorComponent]` - Select content items
- `[UrlSelectorComponent]` - Select URLs or pages
- `[TextInputComponent]` - Text input
- `[CheckBoxComponent]` - Boolean checkbox
- `[DropDownComponent]` - Dropdown selection
- `[ExcludeFromAiraTranslation]` - Exclude from AI translation
- `[UrlValidationRule]` - Validate URLs
- `[RequiredValidationRule]` - Make field required

### Widget ViewComponent (with Data Retrieval)

```csharp
using System.Linq;
using System.Threading.Tasks;
using DancingGoat.Models;
using DancingGoat.Widgets;
using Kentico.Content.Web.Mvc;
using Kentico.PageBuilder.Web.Mvc;
using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.ViewComponents;

[assembly: RegisterWidget(
    CardWidgetViewComponent.IDENTIFIER, 
    typeof(CardWidgetViewComponent), 
    "{$dancinggoat.cardwidget.title$}", 
    typeof(CardWidgetProperties), 
    Description = "{$dancinggoat.cardwidget.description$}", 
    IconClass = "icon-rectangle-paragraph")]

namespace DancingGoat.Widgets
{
    /// <summary>
    /// Controller for card widget.
    /// </summary>
    public class CardWidgetViewComponent : ViewComponent
    {
        public const string IDENTIFIER = "DancingGoat.LandingPage.CardWidget";

        private readonly IContentRetriever contentRetriever;

        public CardWidgetViewComponent(IContentRetriever contentRetriever)
        {
            this.contentRetriever = contentRetriever;
        }

        public async Task<ViewViewComponentResult> InvokeAsync(CardWidgetProperties properties)
        {
            var image = await GetImage(properties);

            return View("~/Components/Widgets/CardWidget/_CardWidget.cshtml", new CardWidgetViewModel
            {
                ImagePath = image?.ImageFile.Url,
                Text = properties.Text
            });
        }

        private async Task<Image> GetImage(CardWidgetProperties properties)
        {
            var image = properties.Image.FirstOrDefault();

            if (image == null)
            {
                return null;
            }

            var result = await contentRetriever.RetrieveContentByGuids<Image>(
                [image.Identifier],
                HttpContext.RequestAborted
            );

            return result.FirstOrDefault();
        }
    }
}
```

## Form Builder Development

Kentico Xperience provides a Form Builder for creating forms that can be embedded in pages through widgets or editable areas. Forms are created in the admin interface and can be customized with custom form sections.

### Form Architecture Overview

1. **Forms** - Created in Kentico admin (BizForms)
2. **Form Sections** - Custom layout containers for form fields
3. **Form Widget** - System widget for embedding forms in pages (`SystemComponentIdentifiers.FORM_WIDGET_IDENTIFIER`)
4. **Form Submission** - Handled automatically by Kentico with optional email notifications

### Form Section Development

Form sections provide custom layouts for organizing form fields within a form.

#### Form Section File Structure

```
/Components/FormSections/TitledSection/
  TitledSectionProperties.cs
  _TitledSection.cshtml
```

#### Form Section Properties

```csharp
using Kentico.Forms.Web.Mvc;
using Kentico.Xperience.Admin.Base.FormAnnotations;

namespace DancingGoat.Components.FormSections.TitledSection
{
    public class TitledSectionProperties : IFormSectionProperties
    {
        [RichTextEditorComponent(Label = "{$dancinggoat.titledsection.title.label$}")]
        public string Title { get; set; }
    }
}
```

**Key Points:**
- Implement `IFormSectionProperties` interface
- Use form component attributes for configurable properties
- Common attributes: `[RichTextEditorComponent]`, `[TextInputComponent]`, `[CheckBoxComponent]`

#### Form Section Registration

Register in `FormBuilderComponentRegister.cs`:

```csharp
using DancingGoat.Components.FormSections.TitledSection;
using Kentico.Forms.Web.Mvc;

[assembly: RegisterFormSection(
    "DancingGoat.TitledSection", 
    "{$dancinggoat.titledsection.title$}", 
    "~/Components/FormSections/TitledSection/_TitledSection.cshtml", 
    Description = "{$dancinggoat.titledsection.description$}", 
    IconClass = "icon-rectangle-a", 
    PropertiesType = typeof(TitledSectionProperties))]
```

**Registration Parameters:**
- **Identifier** - Unique string identifier for the section
- **Name** - Localized display name (resource string)
- **View Path** - Path to the section view
- **Description** - Localized description (optional)
- **IconClass** - CSS class for icon (optional)
- **PropertiesType** - Type of properties class

#### Form Section View

```razor
@using Kentico.Forms.Web.Mvc
@using DancingGoat.Components.FormSections.TitledSection
@using Kentico.Web.Mvc
@using Kentico.PageBuilder.Web.Mvc

@model FormSectionViewModel<TitledSectionProperties>

@if (!string.IsNullOrEmpty(Model?.Properties?.Title))
{
    <div class="ktc-section-title">
        @Html.Raw(Html.Kentico().ResolveRichText(Model.Properties.Title))
    </div>
}

<div class="row">
    <div class="col-md-12">
        @await Html.Kentico().FormZoneAsync()
    </div>
</div>
```

**Key Elements:**
- Model type: `FormSectionViewModel<TPropertiesType>`
- Use `@await Html.Kentico().FormZoneAsync()` to render form field zone
- Access section properties via `Model.Properties`
- Use `Html.Kentico().ResolveRichText()` for rich text fields

### Form Configuration in Admin

Forms are created and configured in the Kentico admin interface:

1. **Form Builder** - Define form fields using drag-and-drop interface
2. **Form Fields** - Text inputs, dropdowns, checkboxes, file uploads, etc.
3. **Form Sections** - Organize fields using custom or system sections
4. **Validation** - Configure field validation rules
5. **Autoresponder** - Configure email notifications on submission
6. **Submit Actions** - Define actions after form submission

### Form Submission Handling

Form submissions are handled automatically by Kentico:

1. **Validation** - Client-side and server-side validation
2. **Data Storage** - Form data stored in database (BizFormItem)
3. **Email Notifications** - Optional autoresponder emails
4. **Redirect** - Optional redirect after submission
5. **Confirmation** - Display confirmation text or message

### Accessing Form Data Programmatically

#### Retrieving Form Submissions

```csharp
using CMS.OnlineForms;

// Get form info
var formInfo = BizFormInfoProvider.GetBizFormInfo("FormCodeName");

if (formInfo != null)
{
    // Get form items (submissions)
    var formItems = BizFormItemProvider.GetItems(formInfo.FormClassName)
        .OrderByDescending("FormInserted")
        .TopN(10);

    foreach (var item in formItems)
    {
        // Access field values
        var email = item.GetValue("Email", "");
        var message = item.GetValue("Message", "");
    }
}
```

### Form Best Practices

1. **Code Names** - Use clear, descriptive code names (e.g., "ContactUsForm", "NewsletterSubscription")
2. **Validation** - Always implement proper validation rules in admin
3. **Localization** - Use resource strings for all form labels and messages
4. **Security** - Enable CAPTCHA for public forms to prevent spam
5. **Email Notifications** - Configure autoresponders for user confirmation
6. **Data Privacy** - Include consent fields for GDPR compliance
7. **Success Messages** - Provide clear confirmation after submission
8. **Error Handling** - Configure user-friendly error messages

### Form Section Customization

Custom form sections allow you to:
- Add custom HTML structure around form fields

- Create multi-column layouts
- Add custom styling and branding
- Implement conditional visibility logic

## Controller Development

### Controller Registration

Use assembly-level attributes to register routes:

```csharp
using Kentico.Content.Web.Mvc.Routing;

[assembly: RegisterWebPageRoute(
    ArticlePage.CONTENT_TYPE_NAME, 
    typeof(DancingGoatArticleController), 
    WebsiteChannelNames = new[] { DancingGoatConstants.WEBSITE_CHANNEL_NAME }, 
    ActionName = "Article")]
```

### Controller Structure

```csharp
using System.Linq;
using System.Threading.Tasks;
using CMS.DataEngine;
using DancingGoat.Models;
using Kentico.Content.Web.Mvc;
using Kentico.PageBuilder.Web.Mvc.PageTemplates;
using Microsoft.AspNetCore.Mvc;

namespace DancingGoat.Controllers
{
    public class DancingGoatArticleController : Controller
    {
        private readonly IContentRetriever contentRetriever;

        public DancingGoatArticleController(IContentRetriever contentRetriever)
        {
            this.contentRetriever = contentRetriever;
        }

        // Action for listing pages
        public async Task<IActionResult> Index()
        {
            var articlesSection = await contentRetriever.RetrieveCurrentPage<ArticlesSection>(
                HttpContext.RequestAborted
            );

            var articles = await GetArticles(articlesSection);
            var models = articles.Select(ArticleViewModel.GetViewModel);
            var model = ArticlesSectionViewModel.GetViewModel(
                articlesSection, 
                models, 
                articlesSection.GetUrl().RelativePath);

            return View(model);
        }

        // Action for detail page with page template
        public async Task<IActionResult> Article()
        {
            var article = await contentRetriever.RetrieveCurrentPage<ArticlePage>(
                new RetrieveCurrentPageParameters 
                { 
                    IncludeSecuredItems = true, 
                    LinkedItemsMaxLevel = 3 
                },
                HttpContext.RequestAborted
            );

            if (article is null)
            {
                return NotFound();
            }

            var model = ArticleDetailViewModel.GetViewModel(article);

            // Return TemplateResult for pages with page builder templates
            return new TemplateResult(model);
        }

        private async Task<IEnumerable<ArticlePage>> GetArticles(ArticlesSection articlesSection)
        {
            return await contentRetriever.RetrievePages<ArticlePage>(
                new RetrievePagesParameters
                {
                    PathMatch = PathMatch.Children(articlesSection.SystemFields.WebPageItemTreePath),
                    IncludeSecuredItems = true,
                    LinkedItemsMaxLevel = 1
                },
                query => query.OrderBy(OrderByColumn.Desc(nameof(ArticlePage.ArticlePagePublishDate))),
                new RetrievalCacheSettings($"OrderBy_{nameof(ArticlePage.ArticlePagePublishDate)}_{nameof(OrderByColumn.Desc)}"),
                HttpContext.RequestAborted
            );
        }
    }
}
```

### Key Points:
- Use `IContentRetriever` for all content retrieval
- Return `TemplateResult` for pages with page builder templates
- Return standard `View()` for listing/non-templated pages
- Always handle null cases with `NotFound()`
- Use `HttpContext.RequestAborted` cancellation token
- Inject only required services via constructor

## View Models

### View Model Pattern

View models transform content type models into presentation-ready data.

```csharp
using System;
using System.Collections.Generic;
using System.Linq;
using CMS.Websites;

namespace DancingGoat.Models
{
    public record ArticleDetailViewModel(
        string Title, 
        string TeaserUrl, 
        string Summary, 
        string Text, 
        DateTime PublicationDate, 
        Guid Guid, 
        bool IsSecured, 
        string Url, 
        IEnumerable<RelatedPageViewModel> RelatedPages)
        : IWebPageBasedViewModel
    {
        /// <inheritdoc/>
        public IWebPageFieldsSource WebPage { get; init; }

        /// <summary>
        /// Validates and maps <see cref="ArticlePage"/> to a <see cref="ArticleDetailViewModel"/>.
        /// </summary>
        public static ArticleDetailViewModel GetViewModel(ArticlePage articlePage)
        {
            var teaser = articlePage.ArticlePageTeaser.FirstOrDefault();

            var relatedPageViewModels = new List<RelatedPageViewModel>();
            foreach (var relatedPage in articlePage.ArticleRelatedPages)
            {
                relatedPageViewModels.Add(RelatedPageViewModel.GetViewModel(relatedPage));
            }

            return new ArticleDetailViewModel(
                articlePage.ArticleTitle,
                teaser?.ImageFile.Url,
                articlePage.ArticlePageSummary,
                articlePage.ArticlePageText,
                articlePage.ArticlePagePublishDate,
                articlePage.SystemFields.ContentItemGUID,
                articlePage.SystemFields.ContentItemIsSecured,
                articlePage.GetUrl().RelativePath,
                relatedPageViewModels)
            {
                WebPage = articlePage
            };
        }
    }
}
```

### View Model Best Practices:
- Use **record types** for immutability
- Implement `IWebPageBasedViewModel` for web page models

## Content Retrieval Patterns

### IContentRetriever Methods

#### 1. Retrieve Current Page
```csharp
var currentPage = await contentRetriever.RetrieveCurrentPage<HomePage>(
    new RetrieveCurrentPageParameters 
    { 
        LinkedItemsMaxLevel = 4,
        IncludeSecuredItems = true 
    },
    cancellationToken
);
```

#### 2. Retrieve Pages (Web Pages Query)
```csharp
var articles = await contentRetriever.RetrievePages<ArticlePage>(
    new RetrievePagesParameters
    {
        PathMatch = PathMatch.Children(parentPath),
        IncludeSecuredItems = true,
        LinkedItemsMaxLevel = 1
    },
    query => query
        .OrderBy(OrderByColumn.Desc(nameof(ArticlePage.ArticlePagePublishDate)))
        .TopN(10),
    new RetrievalCacheSettings($"Articles_TopN_10", TimeSpan.FromMinutes(5)),
    cancellationToken
);
```

#### 3. Retrieve Content (Reusable Content Query)
```csharp
var cafes = await contentRetriever.RetrieveContent<Cafe>(
    new RetrieveContentParameters { LinkedItemsMaxLevel = 1 },
    query => query
        .InSmartFolder(smartFolderGuid)
        .TopN(3),
    new RetrievalCacheSettings($"InSmartFolder_{smartFolderGuid}_TopN_3", TimeSpan.FromMinutes(5)),
    cancellationToken
);
```

#### 4. Retrieve Content of Reusable Schemas (Products, etc.)
```csharp
var products = await contentRetriever.RetrieveContentOfReusableSchemas<IProductFields>(
    new RetrieveContentOfReusableSchemasParameters
    {
        LinkedItemsMaxLevel = 2,
        IncludeSecuredItems = User.Identity.IsAuthenticated
    },
    query => query
        .Where(p => p.ProductFieldCategory.Any(c => categoryGuids.Contains(c.Identifier)))
        .OrderBy(nameof(IProductFields.ProductFieldName)),
    RetrievalCacheSettings.CacheDisabled,
    cancellationToken
);
```

#### 5. Retrieve Content by GUIDs
```csharp
var image = await contentRetriever.RetrieveContentByGuids<Image>(
    [imageGuid],
    cancellationToken
);

var imageItem = image.FirstOrDefault();
```

### Content Query Extensions
- `.TopN(int)` - Limit results
- `.OrderBy(string fieldName)` - Order ascending
- `.OrderByDescending(string fieldName)` - Order descending
- `.Where(predicate)` - Filter results
- `.InSmartFolder(Guid)` - Filter by smart folder

### Path Matching
```csharp
// Direct children (1 level)
PathMatch = PathMatch.Children(treePath, 1)

// All descendants
PathMatch = PathMatch.Children(treePath)

// Specific page
PathMatch = PathMatch.Single(treePath)
```

## Navigation Implementation

### Navigation Service Pattern

```csharp
using System.Collections.Generic;
using System.Linq;
using System.Threading;
using System.Threading.Tasks;
using CMS.Helpers;
using CMS.Websites;
using CMS.Websites.Routing;
using DancingGoat.Models;
using Kentico.Content.Web.Mvc;

namespace DancingGoat.ViewComponents
{
    public class NavigationService
    {
        private readonly IContentRetriever contentRetriever;
        private readonly IWebPageUrlRetriever webPageUrlRetriever;
        private readonly IWebsiteChannelContext websiteChannelContext;
        private readonly IProgressiveCache progressiveCache;

        public NavigationService(
            IContentRetriever contentRetriever,
            IWebPageUrlRetriever webPageUrlRetriever,
            IWebsiteChannelContext websiteChannelContext,
            IProgressiveCache progressiveCache)
        {
            this.contentRetriever = contentRetriever;
            this.webPageUrlRetriever = webPageUrlRetriever;
            this.websiteChannelContext = websiteChannelContext;
            this.progressiveCache = progressiveCache;
        }

        public async Task<IEnumerable<NavigationItemViewModel>> GetSiteNavigationItemViewModels(
            string languageName, 
            CancellationToken cancellationToken = default)
        {
            return await progressiveCache.LoadAsync(async (cacheSettings, cancellationToken) =>
            {
                // Retrieve navigation items
                var navigationItems = (await contentRetriever.RetrievePages<NavigationItem>(
                    new RetrievePagesParameters
                    {
                        PathMatch = PathMatch.Children(treePath, 1)
                    },
                    query => query.OrderBy(nameof(IWebPageContentQueryDataContainer.WebPageItemOrder)),
                    RetrievalCacheSettings.CacheDisabled,
                    cancellationToken
                )).ToList();

                // Bulk retrieve URLs for all linked pages
                var urls = await webPageUrlRetriever.Retrieve(
                    [.. navigationItems.SelectMany(x => x.NavigationItemLink.Select(y => y.WebPageGuid))],
                    websiteChannelContext.WebsiteChannelName, 
                    languageName, 
                    websiteChannelContext.IsPreview, 
                    cancellationToken);

                return navigationItems.Select(x => new NavigationItemViewModel(
                    x.NavigationItemName,
                    urls[x.NavigationItemLink.First().WebPageGuid].RelativePath
                ));
            },
            new CacheSettings(10, $"NavigationItems_{treePath}_{languageName}"), 
            cancellationToken);
        }
    }
}
```


## Naming Conventions

### Files and Folders
- **Widget folder**: `{WidgetName}Widget/` (e.g., `CardWidget/`)
- **Widget files**: 
  - `{Name}WidgetProperties.cs`
  - `{Name}WidgetViewComponent.cs`
  - `{Name}WidgetViewModel.cs`
  - `_{Name}Widget.cshtml` or `_{Namespace}_{Scope}_{Name}Widget.cshtml`

- **Form section folder**: `{SectionName}/` (e.g., `TitledSection/`)
- **Form section files**:
  - `{Name}SectionProperties.cs`
  - `_{Name}Section.cshtml`

- **Controller**: `{Namespace}{PageType}Controller.cs` (e.g., `DancingGoatArticleController.cs`)
- **View**: `_{PageTemplateName}.cshtml` (e.g., `_Article.cshtml`)

### Identifiers
- **Widget**: `{Namespace}.{Scope}.{Name}` (e.g., `DancingGoat.LandingPage.CardWidget`)
- **Section**: `{Namespace}.{Name}Section` (e.g., `DancingGoat.SingleColumnSection`)
- **Form Section**: `{Namespace}.{Name}Section` (e.g., `DancingGoat.TitledSection`)
- **Page Template**: `{Namespace}.{Name}` (e.g., `DancingGoat.ArticleTemplate`)
- **Content Type**: `{Namespace}.{Name}` (e.g., `DancingGoat.ArticlePage`)
- **Form Code Name**: `{Namespace}{Purpose}` (e.g., `DancingGoatSubscription`, `DancingGoatContactUs`)

### Code Naming
- **Properties classes**: `{Name}Properties`
- **View models**: `{Name}ViewModel`
- **ViewComponents**: `{Name}ViewComponent`
- **Services**: `{Name}Service`
- **Repositories**: `{Name}Repository`

### Constants
- Use `UPPER_SNAKE_CASE` for public constants
- Define in dedicated constants files or as class constants

## Additional Best Practices

### Error Handling
- Always check for null content items
- Return `NotFound()` for missing pages
- Use try-catch for external service calls

### Localization
- Use resource strings for all UI text: `{$resource.string.key$}`
- Never hardcode display text in code or attributes

### Security
- Use `IncludeSecuredItems` parameter based on authentication state
- Check `User.Identity.IsAuthenticated` for secured content

### Performance
- Use `LinkedItemsMaxLevel` to control query depth (default is 0)
- Implement caching for frequently accessed data
- Use bulk URL retrieval (`IWebPageUrlRetriever.Retrieve()`)
- Avoid N+1 queries - retrieve related items in batches

### Page Builder
- Return `TemplateResult` for pages with templates
- Use `Context.Kentico().PageBuilder().EditMode` to detect edit mode
- Implement inline editors for better content editing experience
- Restrict widgets per editable area using `AreaRestrictionHelper`

### Forms
- Always use resource strings for form labels and messages
- Enable CAPTCHA on public forms to prevent spam
- Configure autoresponder emails for user confirmation
- Include consent agreement fields for GDPR compliance
- Use `FormZoneAsync()` helper to render form field zones
- Test form submissions in both preview and live mode
- Validate form data on both client and server side
